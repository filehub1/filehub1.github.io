これらの悩み、ありますよね？

🔍 ファイルが見つからない？

PC、スマホ、タブレットにファイルが散らかっている。必要な時に見つからない。クラウド検索は遅いし、ローカル検索は一台のデバイスしか搜せない。

📱 デバイス間のアクセスが面倒？

スマホでPCのドキュメントを見たいのに、まずはクラウドにアップしないといけない。デバイス間のファイル転送はストレス。

👀 プレビューが手間？

ファイル形式ごとに専用のアプリが必要。PDF、画像、コード…それぞれ違うアプリで開かないといけない！面倒すぎる！

🐢 検索が遅い？

検索結果が出るのを待つのが苦痛。ファイル名を正確覚えていないと搜し出せない тожеっと速い方法はないものか？

FileHubが解決します

⚡ 即時検索

キーワードを入力すると瞬時に結果が表示される。部分一致、曖昧検索、正規表現に対応。

📄 強力なプレビュー

専用アプリダウンロード不要。ブラウザ内で直接プレビュー：PDF、画像、映像、Office文書、コード、Markdown — 全部入り！

🌐 LANアクセス

同じWi-Fi下ならスマホ、タブレット、PCからアクセス可能。QRコードでスマホから即座に開ける。

🔒 プライバシーとセキュリティ

データはすべてローカル保存、クラウドには一切アップしない。LANアクセスはパスワード保護可能 — セキュリティは自分が管理。

🎨 美しいデザイン

ダーク/ライトテーマで環境に者数。キーボードショートカットを習得すれば生産性アップ。

🔌 クロスプラットフォーム

Windowsデスクトップアプリ、Androidアプリ、Dockerサーバー — ユースケースに合わせて選ぼう。全デバイス対応。

今すぐ始めましょう

自分に合うバージョンを選んでね

🖥️ Windows 📱 Android APK 🐳 Docker 📦 npm 💻 VSCode

ダウンロード＆実行

1. GitHub Releasesからダウンロード
2. zipファイルを解凍
3. filehub.exeを実行

Windows 10/11 と WebView2 ランタイムが必要

Releases →ソースコード →

ダウンロード＆インストール

1. GitHub ReleasesからAPKをダウンロード
2. 設定で「未知のアプリをインストール」を有効化
3. APKファイルを開いてインストール

Android 8.0+（API 26+）が必要

Releases →ソースコード →

Docker実行

docker pull exiahuang/filehub-server:latest
docker run -d -p 6543:6543 -v /data:/data exiahuang/filehub-server:latest
# ブラウザで http://localhost:6543 を開く

1コマンドで起動。NAS、ラズパイ、任意のサーバーに対応。

Docker Hub →ソースコード →

インストール＆実行

npm install -g @exiahuang/filehub
filehub-server
# ブラウザで http://localhost:6543 を開く

Node.js 18+ が必要

npm →ソースコード →

インストール＆使い方

1. VSCode Marketplaceからインストール（"FileHub"で検索）
   または: code --install-extension filehub-vscode.vsix
2. Ctrl+Shift+F でFileHubを開く
3. 設定でインデックスディレクトリを設定

VSCode 1.85+ と Node.js 18+ が必要
